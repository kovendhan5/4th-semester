Literals

1. Numerical Literals:

A numerical literal is a literal value that represents a number.
Numeric literals in JavaScript represent numeric values. These can be integers or floating-point numbers.
Here are examples of numeric literals:

let integer = 42; // Integer literal
let float = 3.14; // Floating-point literal
let negative = -10; // Negative integer literal
let exponential = 2e3; // Exponential notation: 2 * 10^3

Numeric literals can be expressed in decimal (base 10), hexadecimal (base 16), octal (base 8), or binary (base 2) formats:

let decimal = 42; // Decimal literal (base 10)
let hexadecimal = 0x2A; // Hexadecimal literal (base 16)
let octal = 0o52; // Octal literal (base 8)
let binary = 0b101010; // Binary literal (base 2)


2. String Literals

String literals in JavaScript represent sequences of characters enclosed in single (' ') or double (" ") quotes.
Here are examples of string literals:

let singleQuotes = 'Hello, world!'; // Single quotes
let doubleQuotes = "JavaScript"; // Double quotes
let stringWithEscape = 'I\'m happy'; // Using escape character (\)
let multiLine = `This is
a multi-line
string`; // Template literals (backticks)


JavaScript allows you to use escape characters within string literals to represent characters that are difficult or impossible to type directly into the string. Common escape characters include:

\n: Newline
\t: Tab
\': Single quote
\": Double quote
\\: Backslash

 Template literals, introduced in ECMAScript 6 (ES6), provide a more convenient way to create strings. They are enclosed in backticks (```) and allow for interpolation of variables and expressions using ${}:

let name = "Alice";
let greeting = `Hello, ${name}!`; // Interpolation with template literals

Template literals also support multi-line strings without the need for explicit newline characters. This makes them especially useful for creating more readable code when dealing with lengthy strings or HTML templates.

3. Object literals

Object literals in JavaScript are used to define objects by specifying key-value pairs within curly braces {}.
Each key-value pair represents a property of the object. Here's an example of an object literal:

let person = {
name: "John", // Property: name, Value: "John"
age: 30, // Property: age, Value: 30
isAdmin: false // Property: isAdmin, Value: false
};


In this example, person is an object with three properties: name, age, and isAdmin. The keys (name, age, isAdmin) are called property names, and the corresponding values ("John", 30, false) are called property values.

Object literals can contain various types of values, including strings, numbers, booleans, arrays, other objects, and even functions:

let book = {
title: "The Great Gatsby",
author: {
name: "F. Scott Fitzgerald", dob: 1896,
nationality: "American"
},
pages: 218,
isBestseller: true,
genres: ["Fiction", "Classics"],
printInfo: function() {
console.log(`${this.title} by ${this.author.name}`);
}
};

You can access the properties of an object using dot notation (objectName.propertyName) or bracket notation (objectName["propertyName"]):

console.log(person.name); // Output: "John"
console.log(book["title"]); // Output: "The Great Gatsby"

4. Array literals

Array literals in JavaScript are used to create arrays, which are ordered collections of values. Arrays are defined by enclosing their elements within square brackets [], separated by commas. Here's an example of an array literal:

let mixedArray = [42, "Hello", true, {name: "John"}, [1, 2, 3]];
You can access elements of an array using square bracket notation with the index of the element you want to access:

console.log(mixedArray[0]);
console.log(mixedArray[1]);
console.log(mixedArray[2]);

5. RegExp literals

Regular expression literals in JavaScript are used to define patterns that can be used to perform pattern matching and search operations on strings. Regular expressions are enclosed within forward slashes / / and may contain optional flags that control how the pattern matching is performed. Here's an example of a regular expression literal:

let pattern = /[a-z]+/;


In this example, pattern is a regular expression that matches one or more lowercase letters (a through z). Regular expressions can include a variety of special characters and quantifiers to define more complex patterns.
